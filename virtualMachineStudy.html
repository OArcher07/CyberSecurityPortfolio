Virtual Machine Study

Here are some of my notes and resources regarding the understanding of virtual machines.


What is a Virtual Machine?

  1. Definition: A virtual machine the functions as a virtual computer system with its own CPU, memory, network interface, and storage, created on a physical hardware system.
  2. Hypervisor (virtual machine monitor VMM)
    1. Software that creates and runs VMs
    2. Allows computer to support multiple guest VMs by virtually sharing its resources, such as memory and processing.
    3. Examples: Kernel-based VM, Red Hat software, VMware
    4. Manages hardware and seperates physical resources from virtual environments
    5. Schedules request to access resources so the VMs operating system can access the shared pool of physical resources.
    6. Types of Hypervisors
      1. Definition: Types of software that allows us to run an operating system within another operating system.
      2. Bare metal
        1. VM resources schedule directly to hardware by hypervisor (KVM, LinuxKernel)
      3. Hosted
        1. Schduled against host operating system, then executed against hardware (VMware workstation, Oracle VirtualBox)

Why use a Virtual Machine?

  1. Server Consolidation
    1. Place many virtual servers onto each physical server to improve hardware utilization.
  2. Keeps you from needing to purchase additional physical resources
    1. Enable failover and redundancy virtually
      1. Failover: The ability to switch automatically to a reliable backup system
  3. Environment that is isolated from the rest of the system
    1. Won't interfere with anything else running on the host hardware
    2. Will contain damage if affected by malware, keeping the host system unaffected

What is Proxy/Proxy Chaining?

  1. Using a middle-man
    1. Attacks are sent through proxy servers
      1. The more proxy, the better hidden you are (proxy chaining)
